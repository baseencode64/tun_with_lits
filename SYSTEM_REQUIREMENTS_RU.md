## Требования к сетевому интерфейсу

### TUN-устройство

Приложению требуется поддержка **TUN (туннельного) устройства** в системе. Это виртуальный сетевой интерфейс ядра, работающий на уровне 3 (IP-пакеты).

| Требование              | Описание                                         |
| ----------------------- | ------------------------------------------------ |
| **Модуль TUN**          | Должен быть доступен и загружен (`modprobe tun`) |
| **Права доступа**       | Требуется root или capability `CAP_NET_ADMIN`    |
| **Права маршрутизации** | Возможность изменять таблицы маршрутизации       |
| **Имя интерфейса**      | Автогенерация (обычно `tun0`, `tun1` и т.д.)     |
| **IPv4 адрес**          | `192.18.0.1/32` (по умолчанию, настраивается)    |
| **IPv6 адрес**          | `fd00:dead:beef::1/64` (при включённом `--ipv6`) |
| **MTU**                 | 1500 байт (по умолчанию)                         |

### Требуемые сетевые capabilities (Linux)

```bash
# Минимальные capabilities (рекомендуется)
sudo setcap cap_net_raw,cap_net_admin,cap_net_bind_service+eip /usr/local/bin/goxray

# Эти capabilities дают:
# - cap_net_raw: доступ к RAW сокетам (ping, мониторинг трафика)
# - cap_net_admin: настройка сетевых интерфейсов (TUN, маршруты, iptables)
# - cap_net_bind_service: привязка к привилегированным портам (< 1024)
```

### Описание Linux Capabilities

| Capability             | Требуется для                                                    |
| ---------------------- | ---------------------------------------------------------------- |
| `CAP_NET_ADMIN`        | Создание TUN интерфейса, добавление/удаление маршрутов, iptables |
| `CAP_NET_RAW`          | Чтение /proc/net/dev, операции с RAW сокетами                    |
| `CAP_NET_BIND_SERVICE` | Привязка SOCKS прокси к любому порту                             |

### Используемые сетевые порты

| Порт              | Протокол | Направление          | Назначение                              |
| ----------------- | -------- | -------------------- | --------------------------------------- |
| **Динамический**  | TCP      | Исходящий            | Подключение к XRay серверу (обычно 443) |
| **Динамический**  | TCP      | Входящий (localhost) | SOCKS5 прокси для tun2socks             |
| **Настраиваемый** | TCP      | Входящий             | Prometheus метрики (`--metrics-port`)   |

### Операция → Требуемое разрешение

| ----------------------- | ----------------------- | ---------------------- |
| **Запуск программы**    | `sudo goxray`           | `goxray`               |

## Требования к файрволу / политике безопасности

При работе под строгим файрволом, SELinux или AppArmor:

1. **Исходящий TCP** к порту XRay сервера должен быть разрешён (обычно 443)
2. **Создание TUN устройства** должно быть разрешено
3. **Изменение таблиц маршрутизации** должно быть разрешено
4. **Изменение правил iptables** (для DNS защиты) должно быть разрешено

### SELinux (Red Hat / Fedora)

```bash
# Разрешить доступ к TUN устройству
sudo setsebool -P domain_can_mmap_files 1

# Или создать собственную политику для goxray
```

### AppArmor (Debian / Ubuntu)

Профили AppArmor обычно разрешают TUN и сетевые операции по умолчанию. При проблемах:

```bash
# Проверить, блокирует ли профиль
sudo journalctl | grep "apparmor"

# Временно отключить для goxray
sudo aa-complain /usr/local/bin/goxray
```

## Требования к интернет-подключению

### Требования для начального подключения

1. **DNS разрешение** должно работать для имени хоста XRay сервера
2. **TCP доступность** XRay сервера IP:порт
3. **Доступ к raw URL** (HTTPS) для загрузки списка серверов (при использовании `--from-raw`)
4. **Отсутствие прозрачного прокси**, перехватывающего XRay трафик на порту 443

### Требования к URL списка серверов

| Протокол                  | Статус                                   |
| ------------------------- | ---------------------------------------- |
| **HTTPS** (рекомендуется) | ✅ Полная поддержка                      |
| **HTTP**                  | ✅ Поддерживается (не рекомендуется)     |
| **Локальный файл**        | ✅ Поддерживается (file:///путь/к/файлу) |

## Резюме

Для запуска GoXRay VPN Client необходимо:

1. ✅ **Linux x86_64 или arm64** (или macOS)
2. ✅ **Поддержка TUN устройства** (`modprobe tun`)
3. ✅ **Root или capabilities** для сетевых операций
4. ✅ **~30 МБ RAM** и **< 5% CPU**
5. ✅ **Исходящий TCP** доступ к XRay серверу
6. ✅ **~50 МБ дискового пространства**

Все остальные требования опциональны и зависят от вашей конфигурации (IPv6, DNS защита, метрики и т.д.).
